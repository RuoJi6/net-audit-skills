# 漏洞统一分级标准

所有 .NET 审计 Skill 共享此分级体系，确保漏洞评级和编号一致。该标准对齐 `java-audit-skills/skills/java-shared/SEVERITY_RATING.md`。

## 1. 严重等级计算公式

```text
严重等级 = f(可达性, 影响范围, 利用复杂度)

Score = R * 0.40 + I * 0.35 + C * 0.25
CVSS 映射 = Score / 3.0 * 10.0
```

## 2. 三维评分标准

### 可达性 (Reachability, R)

| 分值 | 级别 | 判定标准 |
| --- | --- | --- |
| 3 | 高 | 无需认证，HTTP 直接可达 |
| 2 | 中 | 需要普通用户认证 |
| 1 | 低 | 需要管理员权限或内网访问 |
| 0 | 无 | 代码不可达 / 死代码 |

### 影响范围 (Impact, I)

| 分值 | 级别 | 判定标准 |
| --- | --- | --- |
| 3 | 高 | RCE / 任意文件写入 / 完全数据泄露 / 系统沦陷 |
| 2 | 中 | 敏感数据泄露 / 越权操作 / 部分文件读取 |
| 1 | 低 | 有限信息泄露 / 低影响配置读取 |
| 0 | 无 | 无实际安全影响 |

### 利用复杂度 (Complexity, C)

反向计分，越容易利用分值越高。

| 分值 | 级别 | 判定标准 |
| --- | --- | --- |
| 3 | 低复杂度 | 单次请求即可利用，无前置条件 |
| 2 | 中复杂度 | 需要构造特殊 payload 或多步操作 |
| 1 | 高复杂度 | 需要特定环境 / 竞态条件 / 链式利用 |
| 0 | 不可利用 | 有效防护，无法绕过 |

## 3. 等级映射

| 前缀 | Score 区间 | CVSS 3.1 | 含义 |
| --- | --- | --- | --- |
| C | 2.70 - 3.00 | 9.0 - 10.0 | 可直接导致系统沦陷 |
| H | 2.10 - 2.69 | 7.0 - 8.9 | 可造成重大损害 |
| M | 1.20 - 2.09 | 4.0 - 6.9 | 可造成一定损害 |
| L | 0.10 - 1.19 | 0.1 - 3.9 | 安全加固项 |

## 4. 漏洞编号规范

```text
格式: {等级前缀}-{漏洞类型}-{序号}

示例:
  H-AUTH-001     高危鉴权绕过 / 越权
  M-SQL-002      中危 SQL 注入
  L-FILE-001     低危文件读取问题
```

类型代码：

| 类型代码 | 对应 Skill | 漏洞类型 |
| --- | --- | --- |
| AUTH | net-auth-audit | 鉴权绕过 / 未授权访问 / 越权 |
| SQL | net-sql-audit | SQL 注入 |
| FILE | net-file-read-audit | 任意文件读取 |
| UPLOAD | net-file-upload-audit | 文件上传 |
| XXE | net-xxe-audit | XXE 外部实体注入 |
| VULN | net-vuln-scanner | 组件漏洞 |

## 5. 可利用性标注与评级关系

| 可利用性 | 对可达性 R 的影响 | 对利用复杂度 C 的影响 |
| --- | --- | --- |
| 已确认可利用 | 按实际认证需求打分 | 按实际复杂度打分 |
| 待验证 | 按实际认证需求打分 | 降 1 级 |
| 不可利用 | R = 0 | C = 0 |
| 环境依赖 | 降 1 级 | 降 1 级 |

## 6. 鉴权类典型评级参考

| 场景 | R | I | C | Score | CVSS | 编号示例 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| 无认证访问管理接口 | 3 | 2 | 3 | 2.65 | 8.8 | H-AUTH-001 |
| 普通用户越权访问其他用户敏感数据 | 2 | 2 | 3 | 2.25 | 7.5 | H-AUTH-002 |
| 跨租户访问业务数据 | 2 | 3 | 2 | 2.35 | 7.8 | H-AUTH-003 |
| 仅认证未做角色校验，影响普通管理功能 | 2 | 2 | 2 | 2.00 | 6.7 | M-AUTH-001 |
| `[AllowAnonymous]` 放行疑似敏感接口但需业务确认 | 3 | 1 | 1 | 1.80 | 6.0 | M-AUTH-002 |
| Session 超时过长 | 1 | 1 | 1 | 1.00 | 3.3 | L-AUTH-001 |

## 7. 报告输出字段

每个漏洞详情必须包含：

| 项目 | 信息 |
| --- | --- |
| 漏洞编号 | `{C/H/M/L}-AUTH-{序号}` |
| 严重等级 | Critical / High / Medium / Low + CVSS |
| 可达性 (R) | 0-3 + 判定理由 |
| 影响范围 (I) | 0-3 + 判定理由 |
| 利用复杂度 (C) | 0-3 + 判定理由 |
| 可利用性 | 已确认 / 待验证 / 不可利用 / 环境依赖 |
| 位置 | 类.方法 + 文件路径:行号，或反编译产物路径 |
